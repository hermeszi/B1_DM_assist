## Database Schema

```sql
-- Campaigns
CREATE TABLE campaigns (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    name TEXT NOT NULL,
    setting TEXT,
    tone TEXT,              -- 'dark', 'whimsical', 'heroic', etc.
    system TEXT DEFAULT 'DnD 5e',
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- NPCs encountered
CREATE TABLE npcs (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    campaign_id INTEGER,
    name TEXT NOT NULL,
    role TEXT,              -- 'ally', 'villain', 'merchant', 'quest-giver'
    notes TEXT,             -- DM's notes about this NPC
    is_alive BOOLEAN DEFAULT 1,
    FOREIGN KEY (campaign_id) REFERENCES campaigns(id)
);

-- Locations
CREATE TABLE locations (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    campaign_id INTEGER,
    name TEXT NOT NULL,
    type TEXT,              -- 'tavern', 'dungeon', 'forest', 'city'
    description TEXT,
    FOREIGN KEY (campaign_id) REFERENCES campaigns(id)
);

-- Sessions
CREATE TABLE sessions (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    campaign_id INTEGER,
    session_number INTEGER,
    date DATE,
    summary TEXT,           -- what happened this session
    FOREIGN KEY (campaign_id) REFERENCES campaigns(id)
);

-- AI Generations (the core table - stores what was generated and context)
CREATE TABLE generations (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    session_id INTEGER,
    tier INTEGER,           -- 1, 2, or 3
    generation_type TEXT,   -- 'loot', 'encounter', 'dialogue', 'description', 'twist'
    context_prompt TEXT,    -- what the DM described
    ai_output TEXT,         -- what the AI generated
    dm_reaction TEXT,       -- what happened after (optional)
    is_favorite BOOLEAN DEFAULT 0,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (session_id) REFERENCES sessions(id)
);
```

## AI Prompt Design (The Key Differentiator)

This is what makes your app different from a random table:

```python
SYSTEM_PROMPT = """You are an AI Dungeon Master's Assistant for a {system} campaign.

CAMPAIGN CONTEXT:
- Setting: {setting}
- Tone: {tone}
- Current Location: {location}
- Active NPCs: {npcs_summary}
- Recent Events: {recent_events}

DM'S CURRENT SITUATION: {dm_input}

Generate a {generation_type} that:
1. Is consistent with the established world and NPCs
2. Surprises the DM while being narratively coherent
3. Includes a twist or complication (not just predictable outcomes)

For loot: tie it to the enemy or location's theme.
For encounters: reference recent events or NPC motivations.
For NPC dialogue: maintain character voice and hidden motives.
For plot twists: subvert expectations but don't break established lore.

Output as JSON:
{
  "title": "Short punchy title",
  "content": "The full generated content for the DM to read or paraphrase",
  "mechanic": "Any dice checks, DCs, or game mechanics if applicable",
  "secret": "Something the DM knows but players don't (optional twist)",
  "connects_to": "How this ties to existing NPCs or events"
}
"""
```

This demonstrates **stateful prompting** — the AI maintains narrative continuity. This is a step beyond simple request-response and shows sophisticated AI engineering.
