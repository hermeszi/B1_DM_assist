## 🗄️ Data Model

```sql
-- campaigns table
CREATE TABLE campaigns (
    id INTEGER PRIMARY KEY,
    name TEXT NOT NULL,
    setting TEXT,           -- "High fantasy medieval", "Steampunk noir", etc.
    tone TEXT,              -- "dark", "heroic", "comedic", "gritty"
    current_arc TEXT,       -- summary of current storyline
    world_notes TEXT,       -- freeform world-building notes
    created_at TEXT DEFAULT (datetime('now'))
);

-- party_members table
CREATE TABLE party_members (
    id INTEGER PRIMARY KEY,
    campaign_id INTEGER REFERENCES campaigns(id),
    name TEXT NOT NULL,
    race TEXT,
    class TEXT,
    level INTEGER DEFAULT 1,
    background TEXT,        -- brief backstory
    goals TEXT,             -- personal quests, motivations
    traits TEXT             -- personality, flaws, bonds
);

-- npcs table (auto-populated when NPCs are generated/logged)
CREATE TABLE npcs (
    id INTEGER PRIMARY KEY,
    campaign_id INTEGER REFERENCES campaigns(id),
    name TEXT NOT NULL,
    description TEXT,
    role TEXT,              -- "ally", "enemy", "neutral", "unknown"
    location TEXT,
    notes TEXT,
    first_appeared TEXT     -- session reference
);

-- session_log table (the running record)
CREATE TABLE session_log (
    id INTEGER PRIMARY KEY,
    campaign_id INTEGER REFERENCES campaigns(id),
    session_number INTEGER,
    timestamp TEXT DEFAULT (datetime('now')),
    entry_type TEXT,        -- "generation", "dm_note", "event", "npc_met"
    generation_type TEXT,   -- "loot", "encounter", "npc_dialogue", "description", "plot_hook", "wild_card"
    context_given TEXT,     -- what the DM typed as extra context
    content TEXT,           -- the generated or noted content
    accepted INTEGER DEFAULT 1  -- was this used in the game?
);

-- locations table
CREATE TABLE locations (
    id INTEGER PRIMARY KEY,
    campaign_id INTEGER REFERENCES campaigns(id),
    name TEXT NOT NULL,
    description TEXT,
    visited INTEGER DEFAULT 0,
    notes TEXT
);
```

## 🧠 AI Prompt Architecture

This is the **most important part** — the prompt engineering that makes generations context-aware.

### System Prompt (Base)

```python
SYSTEM_PROMPT = """You are a creative Dungeon Master's assistant for a tabletop RPG.
You generate content that is CONTEXTUALLY APPROPRIATE to the ongoing campaign.

RULES:
1. Generated content must reference or connect to existing campaign elements 
   (NPCs, locations, party goals, story arcs) when natural to do so.
2. Not every generation needs a plot connection — sometimes loot is just loot.
   Aim for ~30% of generations to have a story hook.
3. Match the campaign's TONE setting strictly.
4. Content must be appropriate for the party's LEVEL.
5. Results should surprise the DM — you are creating the unexpected moments 
   that make DMing feel like playing.
6. Be specific and vivid, never generic.
7. Return structured JSON as specified per generation type.
"""
```

### Per-Type Prompt Templates

```python
GENERATION_PROMPTS = {
    "loot": {
        "instruction": """Generate a loot find. Return JSON:
{
  "description": "Narrative description of what they find (2-3 sentences, vivid)",
  "items": [{"name": "...", "value": "...", "magical": bool, "notes": "..."}],
  "hidden_hook": "Optional story connection or null"
}
Level-appropriate: low levels = copper/silver/minor items. High levels = gold/gems/magical items.
Sometimes include something personal, mysterious, or connected to the campaign.""",
    },

    "encounter": {
        "instruction": """Generate a random encounter. Return JSON:
{
  "description": "Scene-setting description (2-3 sentences)",
  "creatures": [{"name": "...", "count": N, "cr": "...", "behavior": "..."}],
  "twist": "What makes this encounter different from a standard fight",
  "resolution_options": ["fight", "negotiate", "flee", "other creative option"],
  "connection": "How this might tie to the campaign story, or null"
}
Not every encounter is combat. Social encounters, environmental hazards, 
moral dilemmas, and strange discoveries are all valid.""",
    },

    "npc_dialogue": {
        "instruction": """Generate NPC dialogue/response. The DM will provide 
the situation and what the party said or did. Return JSON:
{
  "npc_name": "Name (use existing NPC if specified, or create new)",
  "demeanor": "How the NPC is feeling/acting",
  "dialogue": "What the NPC says (in character, with speech mannerisms)",
  "body_language": "Physical description of how they react",
  "hidden_motive": "What the NPC actually wants (DM eyes only)",
  "follow_up_options": ["possible direction 1", "possible direction 2"]
}""",
    },

    "description": {
        "instruction": """Generate atmospheric description text that the DM can 
read aloud or paraphrase. Return JSON:
{
  "read_aloud": "Boxed text the DM can read directly to players (3-5 sentences, 
                  sensory details: sight, sound, smell, feel)",
  "dm_notes": "Details only the DM knows about this scene",
  "interactive_elements": ["thing players might investigate 1", "thing 2", "thing 3"]
}""",
    },

    "plot_hook": {
        "instruction": """Generate a plot hook or story development. This should 
advance or complicate the current story arc, or introduce a compelling side quest. 
Return JSON:
{
  "hook": "The event, revelation, or encounter that creates the hook",
  "delivery": "How this information reaches the party",
  "stakes": "Why this matters",
  "connections": ["existing campaign element 1", "element 2"],
  "possible_outcomes": ["if party pursues", "if party ignores"]
}""",
    },

    "wild_card": {
        "instruction": """Generate something completely unexpected but campaign-appropriate.
This is the DM's "I want to be surprised" button. Could be anything: 
a strange event, a callback to something forgotten, a twist, a comedic moment, 
a dramatic revelation, a mysterious omen. Return JSON:
{
  "event": "What happens (vivid, surprising)",
  "type": "comedic|dramatic|mysterious|dangerous|heartwarming|surreal",
  "impact": "How this might affect the session",
  "dm_tip": "Suggestion for how to play this moment"
}""",
    },
}
```

### Context Assembly Function

```python
def build_context_payload(campaign_id, generation_type, extra_context=""):
    """Assemble all relevant campaign context for the AI request."""
    campaign = get_campaign(campaign_id)
    party = get_party_members(campaign_id)
    recent_log = get_recent_session_log(campaign_id, limit=10)
    npcs = get_npcs(campaign_id)
    locations = get_locations(campaign_id)

    context = f"""
CAMPAIGN: {campaign['name']}
SETTING: {campaign['setting']}
TONE: {campaign['tone']}
CURRENT ARC: {campaign['current_arc']}

PARTY MEMBERS:
{format_party(party)}

KNOWN NPCs:
{format_npcs(npcs)}

KNOWN LOCATIONS:
{format_locations(locations)}

RECENT SESSION EVENTS (last 10):
{format_log(recent_log)}

DM'S CURRENT CONTEXT: {extra_context if extra_context else "No additional context provided."}
"""
    return context
```

## 🔑 Key Decisions to Document

| Decision | Options | Choice | Rationale |
|---|---|---|---|
| Single campaign vs multi | Multi-campaign support, single | Single (with ID for future multi) | 2-hour constraint; schema supports multi already |
| Context window management | Send everything, send recent only, summarize | Send recent 10 + full party/NPC list | Balances context quality vs token cost |
| Generation output format | Plain text, Markdown, JSON | JSON → rendered as styled HTML | Structured data allows logging, reuse, and clean display |
| NPC tracking | Manual only, auto-extract | Auto-extract from accepted NPC dialogues + manual | Reduces DM bookkeeping, core value proposition |
| Session persistence | In-memory, localStorage, SQLite | SQLite | Campaign data must persist across sessions; SQLite is already our stack |
